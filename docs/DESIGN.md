# 设计说明

## 边界

本项目是 QimenBot ABI 0.3 动态插件，不修改或重新发布 QimenBot 框架。
管理日志是唯一主统计源；公开性能接口仅在单模型命令中作为独立参考。

## 数据流

```text
New API /api/log/
  -> 后台采集线程
  -> DTO 与 other 解析
  -> SQLite 日志样本
  -> request_id 请求归并
  -> 常用窗口快照
  -> QQ 命令 / Heartbeat 推送

单模型命令
  -> 本地窗口快照
  -> 可选 /api/perf-metrics 补充
  -> 分段消息
```

## 并发模型

插件只启动一个后台线程。该线程独占 SQLite 写连接，并在每轮采集完成后生成不可变
统计快照。FFI 命令和事件回调只读取快照，避免网络或数据库写入阻塞宿主。

单模型广场补充是唯一允许在命令回调中发生的网络访问，使用独立短超时客户端，
并优先读取缓存。任何错误都降级为只返回本地日志结果。

当一个采集窗口超过单轮分页容量时，采集器会二分选择本轮能够完整读取的最大时间
边界，只将游标推进到该边界。后续轮次继续追赶，并沿用重叠窗口处理迟到日志。

## 请求结果归并

New API 会为每次失败渠道尝试写错误日志，并在最终成功时写消费日志。同一请求使用
相同 `request_id`。插件按时间顺序导入并维护请求结果：

1. 错误日志增加尝试数，短期保持 `pending`。
2. 正常消费日志将请求置为 `success`。
3. 消费日志中流结束状态异常时置为 `partial_failed`。
4. 宽限期结束后仍只有错误日志时置为 `failed`。

请求正常率使用请求结果；尝试错误率使用原始样本。两者绝不混用。

`other.admin_info.use_channel` 只解析正整数渠道 ID，最多保留 16 跳并按原顺序记录。
SQLite v2 同时在样本和请求终态保存归一化重试链，异常报告只展示聚合计数。

配置分组用于限定模型报告的展示与总体统计范围。自动重试时无法确认归属的错误始终
保留在“自动路由/未确认”分组，避免配置过滤掩盖真实失败。

## 安全

管理令牌只从环境变量读取。数据库不保存 IP、访问令牌、完整请求内容或原始管理
响应。错误文本经过密钥、URL 查询参数和长标识符脱敏后最多保存 256 字符。
管理请求按 New API 中间件契约使用原值 `Authorization`，不添加 `Bearer` 前缀。

## 热重载

`#[shutdown]` 设置停止标志并通过条件变量打断等待，然后 `join` 工作线程。只有线程
退出后才允许宿主卸载动态库，避免线程继续执行已经卸载的代码。
